Back in Stock: Web Push Notifications

Web push notifications allow your customers to be instantly notified on their devices when products are restocked, increasing engagement and conversions by reaching customers directly on the platforms they use most.

Overview

Web push notifications provide instant device notifications to customers when products become available again. This channel complements email notifications and often achieves higher engagement rates due to immediate visibility.

Key Benefits:

  • Instant device notifications for immediate customer awareness
  • Direct engagement on platforms customers actively use
  • Increased reach beyond email limitations
  • Higher engagement rates compared to traditional email

Setup & Configuration

Enable Web Push Notifications

To enable and configure web push notifications:

Method 1: Direct Navigation

  1. Go to Back in Stock Alerts → Configuration → Notification Channels tab
  2. Toggle "Enable Web Push Notifications" to active
  3. Save your settings

Method 2: Quick Access

  1. Navigate to Back in Stock Alerts → Campaigns
  2. Click "Manage notification channels" button
  3. Toggle "Enable Web Push Notifications" to active

Configure Notification Templates

Once web push is enabled, customize your notification templates:

  1. Go to Back in Stock Alerts → Campaigns
  2. Click the "Customize" button under Web push template (Note: This button only becomes clickable after enabling web push in Step 1)
  3. Configure your template settings and styling

Template Customization

The web push template editor allows complete customization of your notifications:

Primary Settings:

  • Title: Main headline customers see
  • Description: Detailed message text
  • Button Labels: Call-to-action text (e.g., "Shop Now", "View Product")
  • URL Directs: Where customers land when clicking the notification

Message Variables: You can use the same variables and filters in web push notifications as email templates. Reference our Email Template Variables Guide for complete options.

Common Variables:

  • {{ product.title }} - Product name
  • {{ product.url }} - Direct product link
  • {{ shop.name }} - Your store name
  • {{ customer.first_name }} - Customer's first name

Add Your Store Logo

Customize notifications with your brand:

  1. In the template settings panel, locate Logo section
  2. Click Choose file...
  3. Upload a PNG or JPG image of your store logo
  4. Save your changes

Logo Requirements:

  • File format: PNG or JPG
  • Recommended size: 192x192 pixels or larger
  • Keep file size under 1MB for optimal loading

Testing & Preview

Preview Notifications

Use the built-in preview to see how your notification appears:

  • Review layout and text formatting
  • Verify logo placement and sizing
  • Check button placement and styling

Send Test Notifications

For the most accurate preview:

  1. Click Send test push message in template settings
  2. Receive the notification on your device
  3. Test click-through functionality
  4. Verify landing page experience

Testing Tips:

  • Test on multiple device types (mobile, desktop)
  • Check notification appearance in different browsers
  • Verify notifications work with your device's notification settings

Best Practices

Message Content

  • Keep titles concise (under 50 characters)
  • Use action-oriented language ("Back in stock!", "Available now!")
  • Include product name for clarity
  • Create urgency without being pushy

Timing

  • Send notifications during your customers' active hours
  • Consider time zones for international customers
  • Avoid notification fatigue by spacing campaigns appropriately

Design

  • Use high-quality, recognizable logo
  • Maintain consistent brand colors and styling
  • Ensure buttons are clearly actionable
  • Test readability across different devices

Troubleshooting

Notifications Not Appearing

  • Verify customer has enabled browser notifications
  • Check if customer's device allows notifications from your domain
  • Confirm web push is enabled in your account settings

Low Engagement Rates

  • Review message content for clarity and appeal
  • Test different button text and calls-to-action
  • Ensure landing pages are mobile-optimized
  • Consider notification timing and frequency

Technical Issues

  • Verify logo file format and size requirements
  • Check template variable syntax
  • Test notifications across different browsers
  • Contact support if delivery issues persist
